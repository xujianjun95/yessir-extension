🫡 Yes Sir - 隐私与数据安全政策

最后更新时间：2024年4月 | 版本：v1.4

您好！我是「Yes Sir」的开发者。作为一款旨在降低浏览器认知负担、提升管理效率的工具，我们深知网页浏览记录是您极其私密的数据。因此，我们在设计之初就确立了“本地优先、最小化权限、透明传输”的核心原则。

本政策旨在向您清晰解释我们会接触到哪些数据，以及这些数据是如何流转的。

📦 1. 数据存储：绝对的本地化

「Yes Sir」所产生的核心业务数据，均仅存储在您的浏览器本地（Local Storage），我们没有也不会建立任何中央数据库来收集您的信息。这些本地数据包括：

  • 您的界面设置与偏好（如快捷键配置、悬浮窗开关等）。
  • 您配置的私人 DeepSeek API Key（如果您选择了自行配置）。
  • 近期被 AI 处理过的网页标签缓存（用于提升二次打开看板的加载速度）。
  • 插件使用次数统计（仅用于本地看板展示）。

🤖 2. AI 功能与数据传输双轨制

为了实现诸如“AI 聚合”、“语义搜索”和“页面标签提取”等高级功能，我们需要将相关的网页信息（仅限网页的标题 Title、网址 URL 和域名）发送给大语言模型（DeepSeek）。

根据您的配置情况，数据流向分为以下两种模式：

🔒 模式 A：直连模式（自带 API Key）

如果您在插件设置中填入了您自己的 DeepSeek API Key，插件会直接与 api.deepseek.com 通信。这期间，您的请求数据不会经过作者的任何服务器，完全是您与模型提供商之间的点对点通信。

☁️ 模式 B：中转模式（开箱即用免配置）

为了让新用户无需配置即可体验 AI 功能，我们提供了一个官方中转服务。当您使用此模式时：

  • 匿名化限流：插件仅会在本地生成一个随机的设备标识（UUID）用于防滥用限流，该标识无法关联到您的真实身份。
  • 数据阅后即焚：中转服务器仅负责透明转发请求。我们绝不持久化存储、不记录日志、不分析任何您发送的网页数据及模型返回的结果。

🛡️ 3. 权限使用声明

我们在 manifest.json 中申请的权限均有明确且克制的功能用途：

  • tabs / windows / tabGroups：用于读取当前打开的标签页列表，执行跨窗口切换、关闭网页以及自动为您创建和重命名标签组。
  • storage：用于在本地保存您的快捷键、缓存数据及 API Key。
  • sessions：用于实现“后悔药”功能，帮您一键恢复最近关闭的标签页。
  • host_permissions：仅允许与 DeepSeek 官方接口及我们的限流中转服务器通信，防止数据外泄到其他未知节点。

❌ 4. 我们不做什么

  • 不收集用户画像：我们不知道您是谁，也不关心您的浏览偏好。
  • 不植入广告代码：纯净工具，没有埋点，没有第三方分析 SDK（如 Google Analytics）。
  • 不收集页面正文内容：AI 识别仅依赖网页的标题（Title）和链接（URL），绝不会读取您正在浏览的网页正文、密码或聊天记录。

🫡 Yes Sir - Privacy & Security Policy

Last Updated: April 2024 | Version: v1.4

Hello! I am the developer of the "Yes Sir" extension. As a tool designed to reduce cognitive load and improve tab management efficiency, we deeply understand that your browsing history is highly private data. Therefore, from the very beginning, we established the core principles of "Local First, Minimal Permissions, and Transparent Transmission".

This policy aims to clearly explain what data we access and how this data flows.

📦 1. Data Storage: Absolute Localization

The core business data generated by "Yes Sir" is stored exclusively in your browser's Local Storage. We do not and will not build any centralized database to collect your information. This local data includes:

  • Your UI settings and preferences (e.g., shortcut configurations, floating widget toggle).
  • Your personal DeepSeek API Key (if you choose to configure one yourself).
  • Cached page labels and categorizations processed by AI (used to speed up rendering when reopening the dashboard).
  • Usage statistics (used exclusively for local dashboard display).

🤖 2. AI Features & Data Transmission Dual-Track System

To implement advanced features such as "AI Aggregation," "Semantic Search," and "Page Label Extraction," we need to send relevant web page information (strictly limited to Page Title, URL, and Domain) to a Large Language Model (DeepSeek).

Depending on your configuration, the data flow falls into two modes:

🔒 Mode A: Direct Connection (Bring Your Own API Key)

If you enter your own DeepSeek API Key in the extension settings, the extension will communicate directly with api.deepseek.com. During this process, your request data will not pass through any of our servers. It is purely peer-to-peer communication between you and the AI model provider.

☁️ Mode B: Relay Mode (Out-of-the-box, No Configuration)

To allow new users to experience AI features without initial configuration, we provide an official relay service. When using this mode:

  • Anonymized Rate Limiting: The extension generates a random device identifier (UUID) locally solely for preventing abuse and applying daily rate limits. This identifier cannot be linked to your real identity.
  • Burn-After-Reading: The relay server only acts as a transparent proxy. We absolutely do not persistently store, log, or analyze any of the webpage data you send or the results returned by the model.

🛡️ 3. Permission Usage Declaration

The permissions we request in manifest.json have explicit and restrained functional purposes:

  • tabs / windows / tabGroups: Used to read the list of currently open tabs, execute cross-window switching, close web pages, and automatically create and rename tab groups for you.
  • storage: Used to save your shortcuts, cached data, and API Key locally.
  • sessions: Used to implement the "Regret Pill" feature, helping you restore recently closed tabs with one click.
  • host_permissions: Strictly limits communication to the official DeepSeek API and our rate-limiting relay server to prevent data leakage to unknown endpoints.

❌ 4. What We Don't Do

  • No User Profiling: We don't know who you are, nor do we care about your browsing preferences.
  • No Ad Code Injection: This is a pure tool. There are no tracking pixels or third-party analytics SDKs (such as Google Analytics).
  • No Page Content Collection: AI recognition relies solely on the page's Title and URL. It will never read the main content, passwords, or chat history of the pages you are browsing.
